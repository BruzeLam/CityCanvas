# CityCanvas

**架空城市地图绘制器** — 面向城市尺度的地貌与路网规划，用「画地图」的方式搭骨架，而不是 CAD 级工程制图。

适合：架空世界观底图、交通立交草图、城市结构推演、教学演示。  
不追求：精确测绘、微观路口渠化、建筑级建模。

---

## 产品定位

| 你在做什么 | CityCanvas 怎么帮你 |
|-----------|-------------------|
| 先定山、水、城的空间关系 | 刷子地貌 + 河道线，几分钟出底图 |
| 再铺快速路 / 主干 / 次干网 | 五级道路 + 平行线 + 弯道 |
| 需要立交、上跨、下穿 | **标高**分层；同层成路口，异层压盖 |
| 匝道连接异层 / 异级 | **匝道**工具：端点挂接、可分段拉通 |
| 路跨河 | 自动识别穿水段 → 桥 / 隧标记 |
| 存档与分享 | 浏览器本地缓存、可选云端、导出 PNG / SVG / `.md` |

核心机制一句话：

> **同层相交 = 路口；异层相交 = 上跨压盖；匝道用连续标高连接两层，中段相交不织平面路口。**

## 快速开始

```bash
npm install
cp .env.example .env   # 需要登录/云端时再配
npm run dev
```

- 前端：http://localhost:5173  
- API：http://localhost:3000（Vite 代理 `/api`）  
- 也可纯本地模式继续画图（不登录）

## 用户手册

### 1. 界面与基础操作

| 操作 | 说明 |
|------|------|
| **拖图 · H** | 平移画布；滚轮缩放；WASD / 方向键也可平移 |
| **编辑 · V** | 选中要素、拖顶点、改标高 |
| **撤销 · Ctrl/⌘ Z** | 撤销上一步路网/地貌等编辑 |
| 侧栏「已选中」 | 看类型、顶点数、标高芯片；可改标高或删除 |
| 图层开关 | 地貌 / 街区 / 道路 / 铁路 / 河流 / 标注 / 路口节点 / 网格 |
| 地图风格 | 导航色 / 蓝图 / 素描 |

路口节点默认在编辑时显示；勾选「始终显示」可在绘制时也看见。

---

### 2. 地貌（刷子）

工具栏 **地貌** 抽屉：

| 快捷键 | 工具 | 作用 |
|--------|------|------|
| `1` | 陆地 | 刷回陆地 |
| `2` | 水域 | 刷出湖泊/海面（栅格） |
| `3` | 绿地 | 刷出山地/绿地色块 |
| `4` | 橡皮 | **按类型**擦：地貌 / 道路 / 铁路 / 河道线 / 标注 |
| `5` | 河道 | 画河道**中心线**（与水域刷互补） |

开局可用地形种子生成底图；之后用刷子微调。刷子大小在工具栏调节。

---

### 3. 道路等级（1–5）

在 **交通 → 道路** 下用数字键切换：

| 键 | 等级 | 视觉 |
|----|------|------|
| `1` | 快速路 | 橙，最宽 |
| `2` | 主干路 | 黄 |
| `3` | 次干路 | 白 |
| `4` | 支路 | 浅灰 |
| `5` | **匝道** | 细线；配色见下文 |

同层同级相交会自动插入路口节点并编织；延伸到已有端点可首尾合并。

---

### 4. 直线 / 弯道 / 平行

路径模式（道路与铁路共用）：

| 模式 | 怎么画 |
|------|--------|
| **直线** | 自由角度；靠近横/竖时**软吸正交**；按住 **Shift** 关闭软吸；**Alt** 关路网吸附 |
| **弯道** | 从端点/中心线出发可锁切线画劣弧；空白处三点选鼓包侧 |

**平行线**（仅道路）：勾选后一次画出双侧或单侧平行线，可调间距（米）与侧向（双侧 / 左 / 右）。适合双向分隔带。

完成：双击结束当前折线（以界面提示为准）。

---

### 5. 标高（立交的核心）

标高范围 **-3 … +3**，默认 **0 地面**。

| 场景 | 行为 |
|------|------|
| 同层两条路相交 | 织成平面路口，边线并块 |
| 异层相交（如 0 与 +2） | **高层整层压在低层上**（边线在上），不织路口 |
| 绘制时点到已有路 | 工具栏标高会跟到该路所在层 |
| 选中后改标高 | 侧栏芯片或快捷键 `-` / `=`；平行姐妹线会一起改 |

**建议画法（上跨）**  
1. 先画下层（标高 `0`）  
2. 把绘制标高调到 `+1` / `+2`  
3. 再画上层，直接跨过下层 — 应看到上层边线压过下层路面  

若仍像「融成一块」，检查两条路是不是其实同标高，或平行双线只改了一条。

---

### 6. 匝道

匝道用来连接**不同标高**或**不同道路等级**，线宽固定偏细。

**配色规则**

| 首尾挂接情况 | 颜色 |
|--------------|------|
| 两端都没接到主路 | 灰色（默认匝道色） |
| 只接一端，或两端同级 | 该级纯色 |
| 两端接到不同等级 | 沿路径渐变 |

**拉通与相交**

- 匝道与匝道**端点相接** → 合并成一条，并按新的首尾主路重算配色  
- 匝道**中段相交** → **不织平面路口**；按路径插值标高决定谁压谁（连续变坡时交点处谁高谁在上）  
- 弯道采样已做稀疏化；过密旧线在加载/重织时会自动瘦身  

**建议画法**  
用匝道 + 弯道，从下层主路端点/中心线画到上层主路；可分段，只要端点对齐会拉通。

---

### 7. 桥与隧（过河）

道路穿过**水域刷**或**河道线**附近时，穿水段自动加标记：

- **标高 ≥ 0**：桥 — 浅细**实线**边线 + 两岸括号口  
- **标高 &lt; 0**：隧 — 浅细**虚线**边线 + 两岸括号口  

不再使用厚描边或阴影；上跨感靠标高绘制顺序表达。

---

### 8. 铁路

交通抽屉中的轨道细分（数字键）：

| 键 | 类型 |
|----|------|
| `1` | 铁路（普速） |
| `2` | 高铁 |
| `3` | 地铁（可自定义线路色） |
| `4` | 有轨电车 |

同样支持直线/弯道与标高；站点工具仍为预留。

---

### 9. 标注与其它

- **标注**：点选位置写文字  
- 尺度测距、轮渡、站台等：工具栏已占位，尚未开放  

---

### 10. 存档与导出

| 方式 | 说明 |
|------|------|
| 浏览器本地 | 自动写入本地缓存，刷新可续画 |
| 云端（登录后） | 账号存档；需后端与数据库 |
| 导出 PNG | 当前视口出图 |
| 导出 SVG | 矢量备份 |
| 导出 / 打开 `.md` | 地图结构化备份，可换机带回 |

## 开发与部署

### 技术栈（简）

- 前端：Vite + React + TypeScript（Canvas 绘制）  
- 可选后端：Node API + SQLite（账号与云端存档）  

### 环境变量（后端）

| 变量 | 说明 |
|------|------|
| `JWT_SECRET` | JWT 签名密钥（云端必填） |
| `DATABASE_PATH` | SQLite 路径，默认 `/data/citycanvas.db` |
| `PORT` / `HOST` | 服务端口与监听地址 |

### Docker

```bash
docker build -t citycanvas .
docker run -p 3000:3000 -v citycanvas-data:/data \
  -e JWT_SECRET=你的随机密钥 \
  -e NODE_ENV=production \
  citycanvas
```

### Zeabur

单体 Docker：`API + 前端 + SQLite`。绑定 Volume 到 `/data`，并配置 `JWT_SECRET`。

### Vercel

当前仓库含 `vercel.json`，可部署前端静态资源。若需要登录与云端存档，请使用带 API 与持久化卷的 Docker / Zeabur 部署。

## 常见问题

**Q：异层交叉还是并成一块？**  
A：确认两条路标高不同（选中看侧栏）。平行双线改标高会带上姐妹线；硬刷新后再看高层边线是否压过下层。

**Q：匝道一段灰一段黄？**  
A：未接到主路的段是灰；接到主路后会按首尾等级上色。分段画时把端点对齐，会合并成一条并重算颜色。

**Q：匝道互相穿过怎么办？**  
A：中段不织路口，按插值标高压盖。只有端点相接才会并成一条匝道。

**Q：过河看不到桥隧标记？**  
A：确认路心穿过水域或靠近河道线，且已部署含桥隧绘制的版本；标记是浅细线，需放大细看。

**Q：弯道顶点特别多？**  
A：新弯道已稀疏采样；旧密折线在加载或改标高重织时会自动简化。
